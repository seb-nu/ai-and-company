# MECE Framework — Detailed Reference

_MECE (Mutually Exclusive, Collectively Exhaustive) is the structuring standard at MBB firms._

---

## Definition

- **Mutually Exclusive**: No overlap — each item belongs to exactly one category
- **Collectively Exhaustive**: No gaps — all items together cover the full space

MECE is not just a concept. It is a daily check applied to every grouping in a deck:
agenda sections, framework buckets, options presented, bullet sub-points.

---

## MECE vs. Non-MECE Examples

**Topic: Reasons for declining revenue**

❌ Not MECE (overlapping):
- Loss of existing customers
- Poor customer retention
- New competitors entering market
- Pricing issues

_"Loss of existing customers" and "poor customer retention" overlap._

✅ MECE:
- Volume decline (fewer transactions)
- Price decline (lower revenue per transaction)
- Mix shift (more low-value transactions)

_Together these cover 100% of revenue movement: Revenue = Volume × Price × Mix_

## MECE Patterns (Standard Structures)

Use these pre-tested MECE structures before inventing new ones:

### By Value Chain
Procurement → Production → Distribution → Sales → Service

### By Geography
Global → Regional → Country → Local

### By Time
Short-term (0–6 months) → Medium-term (6–18 months) → Long-term (18+ months)

### By Stakeholder
Customers / Employees / Shareholders / Regulators / Partners

### By P&L Line
Revenue → Cost of Goods Sold → Gross Margin → OpEx → EBITDA → Net Income

### By 2×2 Matrix
Any two independent dimensions can create 4 MECE quadrants.
Example: High/low urgency × High/low impact → prioritization matrix.

### Issue Tree (Hypothesis-Driven)
Break a problem top-down until each branch is answerable with data.
Example: "Why is profit declining?" →
- Revenue declining? → Volume? Price? Mix?
- Costs increasing? → Fixed? Variable? One-time?

## Quick MECE Tests

Before finalizing any grouping, run these checks:

1. **Overlap test**: Can any single item belong to more than one category?
   → If yes, categories are not mutually exclusive.

2. **Gap test**: Is there any item relevant to the topic that doesn't fit any category?
   → If yes, categories are not collectively exhaustive.

3. **Summary test**: Can you write one sentence that summarizes the entire group?
   → If not, the items may not be at the same logical level.

4. **Count test**: Do you have 3–5 items?
   → Fewer than 3 may not be collectively exhaustive. More than 5 suggests overlap or insufficient structuring.

## MECE in Slide Design

Apply MECE at every level of a slide:

- **Agenda sections**: Each section should cover a distinct part of the argument
- **Bullet points**: Each bullet should address a distinct sub-point (no restating)
- **Chart categories**: Axis labels and legend items should not overlap
- **Options presented**: Each option should be genuinely distinct (not just reframings of the same choice)

## Common MECE Failures in Consulting Decks

| Failure mode | Example | Fix |
|---|---|---|
| Catch-all bucket | "Other" as a category | Enumerate what's in "other" or restructure |
| Overlapping buckets | "Revenue growth" + "Sales improvement" | Pick one dimension |
| Missing category | SWOT without external opportunities | Complete the framework |
| Mixed levels | Mixing "Asia" and "China" as peer categories | Ensure items are at the same level of aggregation |
